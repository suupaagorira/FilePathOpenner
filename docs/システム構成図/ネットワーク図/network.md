# ネットワーク図 (概要)

FilePathOpenner は基本的にローカルで動作するアプリケーションであり、外部サーバーとの通信は想定していません。

## ネットワークとの関連性
- URL を `Ctrl+E` 等で開いた場合に外部ブラウザ経由でネットワークへアクセスが行われますが、アプリ自体はネットワーク経由の API 呼び出しは行いません。
- したがって、ネットワークトポロジやセキュリティ設定は、アプリケーション単体ではほぼ不要です。

## 考慮点
- 組織内ポリシーやファイアウォールによって URL のアクセス制限がある場合、所望の URL を開けないことがあります。
- スタートアップ登録などのファイル操作はローカルファイルシステムへのアクセスとなるため、ローカルポリシーや権限の確認が必要です。
