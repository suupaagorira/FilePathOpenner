# 管理者マニュアル

## 概要
このドキュメントは、システム管理者や運用担当者向けに FilePathOpenner のインストール、デプロイ、設定、アップデート、配布、Windows スタートアップ登録に関する手順をまとめます。

## 事前要件
- Node.js と npm が導入されていること（開発環境）
- Windows のスタートアップ登録は PowerShell が利用可能であること

## インストール手順（開発環境）
1. リポジトリを取得

```pwsh
git clone <repository-url>
cd FilePathOpenner
```

2. 依存関係をインストール

```pwsh
npm install
```

3. 起動確認

```pwsh
npm start
```

## ビルド/配布
- ポータブルアプリを作成するには次のコマンドを使用します。

```pwsh
npm run dist
```

ビルド成果物は `release/` に格納されます。Electron Builder の設定は `package.json` の `build` セクションで管理されています。

## Windows 環境でのスタートアップ登録
- アプリは `registerStartup`（PowerShell を利用）でショートカットを作成します。
- 管理者権限がない環境、またはセキュリティポリシーによって PowerShell スクリプトの実行が制限されることがあります。

## 設定の配布・変更
- ユーザーごとの設定は `electron-store` に保存されます。グローバルにプロファイルを強制したい場合は、配布時に各ユーザーの設定ファイルを上書きするスクリプトを提供できます。

## ログと監視
- 現時点で本アプリは専用のログ機構を持っていません。必要であれば main プロセスにログ出力（ファイル）を追加してください。

## 注意点・運用上のポイント
- `electron-store` はユーザープロファイルにファイルを作成して設定を保存します。複数ユーザーで利用するシナリオは考慮してください。
- スタートアップの設定/解除は `main.js` の `registerStartupShortcut`, `unRegisterStartupShortcut` 関数が担います。
- 管理配布する場合、`release/` の成果物を展開し、必要に応じてショートカットを配布、またはポリシーで自動実行させる手順を作成してください。
