# ユーザーマニュアル

## 概要
FilePathOpenner は、クリップボードにコピーしたパス（ファイル、フォルダ、URL）を素早く開くための Electron 製アプリです。あらかじめ設定したショートカットキーでクリップボードの内容をパースし、存在するパスや URL を開きます。設定はすべて自動保存されます。

## 対象ユーザー
- Windows でショートカットを使ってパスを瞬時に開きたいユーザー
- クリップボードからのパス処理や開閉操作を自動化したいエンドユーザー

## 起動方法
1. 依存モジュールをインストール（開発環境）

```pwsh
npm install
```

2. アプリを起動

```pwsh
npm start
```

### ポータブルや配布版を使う場合
- `release/` フォルダ内に配布物がある場合は、該当する exe/portable を実行します。

## 設定画面の構成
設定ウィンドウは左側のサイドバーで 6 つのカテゴリに分かれています。

- **使い方**: 3 ステップの使い方と、対応している入力の一覧を表示します。
- **ショートカット**: パスを開く／親フォルダを開くキーを設定します。
- **パスの整形**: 開く前のクリーンアップ設定（空白削除・除去文字・複数行の結合）。
- **変換ルール**: 前提パスと、プレフィックスルール（ID→URL/共有フォルダ変換）。
- **動作テスト**: 実際に開かずに、解析結果をドライランで確認します。
- **一般**: PC 起動時の自動起動（スタートアップ登録）など。

> すべての設定は変更した時点で自動保存され、画面下部にトーストで通知されます。「設定」ボタンはありません。

## メイン機能の使い方
- クリップボードからパスを開く
  - 設定したショートカット（デフォルト `Ctrl+E`）を押すと、クリップボードにあるパスが開きます。URL は外部ブラウザで開かれます。複数行ある場合は、改行ごとにそれぞれ開くか、1 行に結合して開くかを「パスの整形」で切り替えできます。

- 一つ上の階層を開く
  - `Ctrl+Shift+E`（デフォルト）などで、URL やパスの 1 階層上を開きます。

- ショートカットの変更
  - 「ショートカット」パネルで入力欄をクリックし、割り当てたいキーをそのまま押すだけで設定されます（キーレコーダー方式）。
  - `Esc` で入力をキャンセル、`Backspace` で割り当てを解除（無効化）できます。
  - F1〜F24 以外のキーは、Ctrl / Alt / Shift のいずれかとの組み合わせが必要です。
  - 他アプリと競合して登録に失敗した場合は、入力欄の下に警告が表示されます。別の組み合わせに変更してください。

- 前提パス（Base Path）の設定
  - 相対パスを扱う際に先頭に付加される基準パスを設定できます。「参照…」ボタンからフォルダ選択ダイアログで指定することもできます。入力するとその場で結合例のプレビューが表示されます。

- プレフィックスルール
  - コピーした文字列が、設定した「先頭パターン」で始まるとき、その文字列を URL や共有フォルダの判別パス部とみなし、「結合文字列」と連結して開きます。
  - 例 1（URL）: 先頭パターン `DOC-` ／ 結合文字列 `https://intra/docs/` → `DOC-123` をコピーして `Ctrl+E` を押すと `https://intra/docs/DOC-123` を外部ブラウザで開きます。
  - 例 2（共有フォルダ）: 先頭パターン `案件` ／ 結合文字列 `\\server\share\` → `案件A\資料` をコピーすると `\\server\share\案件A\資料` を開きます。
  - 「先頭パターンを除去して結合」にチェックすると、先頭パターン部分を取り除いてから結合します（例: 先頭パターン `id:` ／ 結合文字列 `https://intra/view?id=` で `id:42` → `https://intra/view?id=42`）。
  - ルールは上から順に評価され、最初にマッチしたルールが適用されます。プレフィックスルールにマッチした場合は「前提パス」より優先されます。各ルールカードの「↑」「↓」ボタンで評価順を入れ替えられます。
  - 各ルールカードには、入力中の設定でどのように変換されるかのプレビューが表示されます。
  - 既に `http(s)://` の URL や絶対パスと判定される文字列にはルールを適用しません。

- 動作テスト（開く前の確認）
  - 「動作テスト」パネルにテキストを貼り付けるか、「クリップボードを読み込む」で取り込むと、現在の設定でどう解釈されるかが一覧表示されます。
  - 各行には、URL / ファイル / フォルダ / 「○階層上を開く」（フォールバック）/ 見つからない、のいずれかのバッジが付きます。
  - 「この内容で開く」ボタンで、そのままの内容を実際に開くこともできます。

## 設定項目（GUI）
- クリップボードを開くショートカット: `openShortcut`（既定 `Ctrl+E`）
- 1 階層上を開くショートカット: `openParentShortcut`（既定 `Ctrl+Shift+E`）
- 前後の空白を削除: `trimSpaces`（既定 ON）
- 前後から除去する文字: `removeList`（既定 `<>＜＞()（）[]「」{}｛｝"”'’`）
- 複数行を 1 つのパスに結合: `openAsSinglePath`（既定 ON。改行をすべて結合する）
- 前提パス: `basePath`（相対パス時に付加）
- プレフィックスルール: `prefixRules`（`{ prefix, base, stripPrefix }` の配列。先頭パターン一致時に `base` と結合して開く）

## タスクトレイ動作
- ウインドウの「×」を押してもアプリは終了せず、タスクトレイに常駐します（初回のみ、常駐した旨を通知します）。
- タスクトレイのアイコンをダブルクリック、またはコンテキストメニューの「設定画面を開く」で再表示できます。
- 完全に終了するには、トレイアイコンを右クリックして「終了」を選びます。
- 多重起動は自動的に防止され、既に起動している場合は既存のウィンドウが前面に出ます。

## スタートアップ登録（Windows のみ）
- 「一般」パネルの「PC 起動時に自動で開始」トグルを ON にすると、アプリのショートカットが Windows のスタートアップフォルダに配置されます。OFF で解除します。
- Windows 以外のプラットフォームでは、このトグルは無効化されます。

## よくある操作フロー
1. クリップボードに「C:\\Users\\user\\Documents\\file.txt」をコピー
2. `Ctrl+E` を押す → ファイルが関連アプリで開かれる
3. 相対パスを扱う場合は「前提パス」を設定しておく

## 注意点
- スタートアップ登録は PowerShell を使って `.lnk` ショートカットを作成しているため、管理者権限やポリシーによって失敗することがあります。
- 「複数行を 1 つのパスに結合」を ON にすると、改行が除去されるため複数パスは 1 つとして扱われます。
